# LearnPilot 学习计划

> 目标：从零开始，一步一步成为能独立设计 AI Agent 的开发者。
> 原则：每个阶段只学一件新事，做完就能看到效果。

---

## 学习路线总览

```
阶段 1  →  阶段 2  →  阶段 3  →  阶段 4  →  阶段 5
调用LLM     多轮对话    流式输出    工具调用    Web接口

阶段 6  →  阶段 7  →  阶段 8  →  阶段 9  →  阶段 10
保存数据    第一个Agent  意图路由    结构化输出   知识库检索

阶段 11
跨会话记忆（完整 LearnPilot）
```

每个阶段大约需要 **1～2 小时** 学习 + 动手。

---

## 阶段 1：第一次和 LLM 对话

**你会学到：** 什么是 LLM API、怎么用 Python 调用它、消息格式是什么样的

**你会做出：** 一个 Python 脚本，运行后 AI 会回答你的问题

**代码量：** 约 15 行

**文件：** `learn/01_hello_llm.py`

**核心知识点：**
- `openai` 库是什么，怎么安装
- `client.chat.completions.create()` 的参数含义
- `messages` 列表的格式：每条消息有 `role` 和 `content` 两个字段
- `role` 只有三种：`system`（AI 的人设）、`user`（你说的话）、`assistant`（AI 的回复）

---

## 阶段 2：多轮对话

**你会学到：** 为什么 LLM "记得住"你说过的话，原理是什么

**你会做出：** 在终端里和 AI 持续聊天（像聊天软件一样）

**代码量：** 约 30 行

**文件：** `learn/02_conversation.py`

**核心知识点：**
- LLM 本身没有记忆，每次调用都是全新的
- "记忆"是靠把历史消息全部放进 `messages` 列表实现的
- `while True` 循环实现持续对话
- `input()` 读取用户输入

---

## 阶段 3：流式输出（打字机效果）

**你会学到：** 为什么 ChatGPT 的回答是一个字一个字出现的，怎么实现

**你会做出：** AI 的回答像打字机一样逐字打印出来

**代码量：** 约 20 行

**文件：** `learn/03_streaming.py`

**核心知识点：**
- `stream=True` 参数的作用
- 什么是 `async/await`（异步编程基础）
- `for chunk in stream` 逐块处理数据

---

## 阶段 4：给 AI 一个工具

**你会学到：** 什么是 Function Calling（工具调用），AI 怎么决定用不用工具

**你会做出：** AI 能调用一个"搜索"函数来回答你的问题

**代码量：** 约 60 行

**文件：** `learn/04_tools.py`

**核心知识点：**
- 为什么需要工具（LLM 不知道最新信息、不会算数）
- 工具的定义格式：名字、描述、参数
- AI 的回复里如果有 `tool_calls`，说明它想用工具
- 把工具的执行结果加入消息列表，再让 AI 继续回答
- 这就是 **ReAct 模式** 的核心：Reason（思考）→ Act（行动）→ Observe（观察结果）

---

## 阶段 5：包装成 Web 接口

**你会学到：** 什么是 HTTP API，怎么用 FastAPI 写一个接口

**你会做出：** 用 `curl` 或浏览器就能调用的 `/chat` 接口

**代码量：** 约 40 行

**文件：** `main.py` + `src/api/chat.py`

**核心知识点：**
- 什么是 HTTP（请求和响应）
- FastAPI 的基本用法：`@app.post("/chat")`
- 什么是 Pydantic（自动校验输入格式）
- 用 `uvicorn` 启动服务
- 用 `curl` 测试接口

---

## 阶段 6：把对话存入数据库

**你会学到：** 什么是数据库，为什么需要它，怎么用 SQLite 存数据

**你会做出：** 对话记录在服务重启后依然存在，可以查看历史

**代码量：** 约 50 行新增

**文件：** `src/database.py` + `src/models/message.py`

**核心知识点：**
- 什么是 SQLite（一个文件就是一个数据库）
- SQLAlchemy 是什么（Python 操作数据库的工具）
- ORM 是什么（把 Python 类映射成数据库表）
- 异步数据库操作的基本模式

---

## 阶段 7：第一个真正的 Agent（LangGraph 入门）

**你会学到：** 什么是"图"，LangGraph 的核心概念

**你会做出：** 把之前的对话功能改造成 LangGraph 图，功能不变，但结构升级了

**代码量：** 约 60 行

**文件：** `src/agent/graph.py` + `src/agent/state.py`

**核心知识点：**
- 什么是图（Graph）：节点（做事情的函数）+ 边（连接顺序）
- 什么是 State（图的"共享记事本"）
- 节点就是普通的 Python 函数，接收 State，返回更新后的 State
- START 和 END 是特殊节点

---

## 阶段 8：意图识别与路由

**你会学到：** Agent 怎么"决策"，根据用户意图走不同的路

**你会做出：** Agent 能区分"普通聊天"和"制定学习计划"，走不同的处理分支

**代码量：** 约 50 行

**文件：** `src/agent/nodes/intent.py`

**核心知识点：**
- 什么是条件边（Conditional Edge）
- 怎么用 LLM 做分类（意图识别）
- 图的分支结构

---

## 阶段 9：结构化输出

**你会学到：** 怎么让 LLM 输出可以被程序处理的 JSON，而不是自由文本

**你会做出：** 用户输入学习目标，AI 输出一份结构化的学习计划（有阶段、知识点、时长）

**代码量：** 约 40 行

**文件：** `src/agent/nodes/planner.py` + `src/schemas/plan.py`

**核心知识点：**
- `with_structured_output(PydanticModel)` 的用法
- 结构化输出 vs 自由文本的区别
- 为什么工程上必须用结构化输出

---

## 阶段 10：知识库检索（RAG）

**你会学到：** 什么是 Embedding（向量），什么是 RAG，怎么让 AI 基于你的资料回答问题

**你会做出：** 上传一段文字，AI 能基于这段文字回答相关问题

**代码量：** 约 70 行

**文件：** `src/rag/store.py` + `src/agent/nodes/retriever.py`

**核心知识点：**
- 什么是 Embedding（把文字变成数字向量）
- 为什么能用向量做"语义搜索"
- Chroma 向量数据库的基本用法
- RAG 的完整流程：上传 → 向量化 → 存储 → 检索 → 注入 Prompt

---

## 阶段 11：跨会话记忆（完整 LearnPilot）

**你会学到：** Agent 怎么"记住"你是谁、上次学到哪里了

**你会做出：** 完整的 LearnPilot：关掉服务再开启，AI 还记得你上次的学习进度

**代码量：** 约 50 行

**文件：** `src/agent/memory.py`

**核心知识点：**
- LangGraph Checkpointer（图状态快照）
- Thread ID（区分不同用户/会话）
- 记忆 vs 历史消息的区别

## 学习建议

1. **不要跳步骤** — 每个阶段都是下一个阶段的基础
2. **先跑通，再看懂** — 先把代码跑起来，看到效果，再回头理解原理
3. **多改一改** — 每个阶段结束后，试着改改代码，看看会发生什么
4. **阶段 4 是关键** — 工具调用是 Agent 的本质，搞懂这个，后面就顺了
5. **阶段 7 是分水岭** — LangGraph 是本项目的核心框架，第一次接触可能需要多读几遍
